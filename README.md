# じゅんちゃん麻雀

人間1人 vs CPU3人で遊べる、ブラウザ向け4人リーチ麻雀Webアプリを段階的に開発するプロジェクトです。

## MVP-0.1

最初のMVPでは、和了判定なしで「4人卓が流局まで正しく進む」状態を目標にします。

### MVP-0.1で作るもの

- 静的HTML/CSS/JavaScriptのみ
- 外部ライブラリなし
- GitHub Pagesで公開可能
- 牌136枚の生成
- 王牌14枚の分離
- 通常山122枚の作成
- シャッフル
- 4人全員に13枚配牌
- 親が最初に1枚ツモして14枚になる
- 人間が手牌から1枚捨てられる
- CPU3人がランダムに捨てる
- 手番が東、南、西、北、東の順で回る
- 通常山が0枚になったら流局
- 最低限の4人卓UI
- スマホ幅でも人間の手牌を押せる
- localStorageに対局開始数、流局数、最終プレイ日時を保存
- ブラウザで動く簡易テストランナー

### MVP-0.1では作らないもの

- 和了判定
- 役判定
- 点数計算
- チー、ポン、カン
- リーチ
- フリテン
- ドラ効果
- 裏ドラ
- 本格CPU AI
- 東風戦、半荘戦の完全進行

## 実行方法

MVP本体は後続ステップで実装します。実装後は `index.html` をブラウザで開くだけで動く構成にします。

テストは `tests/test-runner.html` をブラウザで開いて実行する方針です。

## 開発方針

- `src/game/` 以下はDOMに触らない
- UIは `gameState` を受け取って描画する
- 操作は `dispatchAction(state, action)` に集約する
- UIはstateを直接変更しない
- type="module" 前提
- 外部ライブラリなし
- pushは確認後に行う
