# MVP-0.1 Design Review

## 参照情報源

- [W3C WAI: Older Users and Web Accessibility](https://www.w3.org/WAI/older-users/)
  - 高齢ユーザーは視力、細かな操作、集中、短期記憶の低下がWeb利用に影響しやすい。
  - 高齢者向けの配慮は、障害のあるユーザー向けアクセシビリティと重なる部分が大きい。
- [WCAG 2.2](https://www.w3.org/TR/WCAG22/)
  - 通常テキストは4.5:1以上、大きい文字は3:1以上のコントラストを目安にする。
  - テキストを200%拡大しても内容や機能が失われないことを目指す。
  - タッチ対象サイズやフォーカス表示を考慮する。
- [Japanese Mahjong - arranged discards](https://en.wikipedia.org/wiki/Japanese_mahjong)
  - 日本麻雀では捨て牌を各プレイヤー前に順序立てて並べる慣習がある。
  - UIでも捨て牌の所属と順序を見せる価値が高い。
- [Mahjong general play](https://en.wikipedia.org/wiki/Mahjong)
  - 13枚手牌、ツモで14枚、打牌で13枚に戻る基本進行を確認した。

## MVP-0.1向けデザイン方針

- 最優先は「今だれの番か」「自分は何を押せばいいか」「流局したか」が一目で分かること。
- 人間の手牌は常に最も大きく、押しやすく表示する。
- CPU情報は簡略化し、手牌は枚数表示だけでよい。
- 牌ボタンは最低44px以上、MVP-0.1では通常50px前後を基準にする。
- 大きい牌モードでは人間手牌を60px以上に拡大する。
- 色だけで手番を示さず、「手番」ラベル、枠線、背景差を併用する。
- スマホは縦画面優先とし、横画面専用UIにはしない。
- 捨て牌は各プレイヤーの近くに小さく並べる。
- MVP-0.1では詳細演出より、現在状態と操作可能状態の明確さを優先する。

## 画面レイアウト案

- 上部
  - タイトル
  - 新規局開始ボタン
  - 大きい牌モード切替ボタン
- 中央
  - 状態文
  - 通常山残り枚数
  - 王牌枚数
  - ドラ表示牌
- 上、左右
  - CPU席
  - CPU手牌枚数
  - CPU捨て牌
  - 手番ラベル
- 下部
  - 人間席
  - 人間の手牌
  - 人間の捨て牌
- 最下部
  - 対局開始数
  - 流局数
  - 最終プレイ日時

## スマホ表示の優先順位

1. 人間の手牌
2. 現在手番と操作可能状態
3. 山残り枚数
4. 流局メッセージ
5. 自分の捨て牌
6. CPUの捨て牌
7. 王牌とドラ表示牌
8. stats

## 大きい牌モード仕様

- 人間の手牌を最優先で拡大する。
- 通常モードの人間手牌は幅46pxから50px程度を基準にする。
- 大きい牌モードの人間手牌は幅60pxから64px程度を基準にする。
- 13枚から14枚が画面幅に収まらない場合は横スクロールを許容する。
- 牌を縮めすぎて押しにくくするより、横スクロールで操作性を守る。
- CPU手牌は枚数表示だけにして、画面を圧迫しない。
- 文字も読みやすく拡大するが、画面全体を過度に巨大化しない。

## おばあちゃん向け配慮

- 小さい薄文字を避ける。
- 「CPU処理中」「あなたの番です」「流局しました」など、短く具体的な文言にする。
- 押せない牌はdisabledにして、見た目でも押せないことを分かるようにする。
- 誤タップ防止として、牌同士に少し隙間を確保する。
- CPUの自動進行が速すぎると不安になるため、300msから600ms程度のdelayを想定する。
- 色だけに頼らず、文字と形でも状態を伝える。
- 画面内の情報量を増やしすぎない。

## Agent Bへの具体的改善指示

- 人間の手牌ボタンは最低 `min-width: 50px; min-height: 66px;` を維持する。
- 大きい牌モードでは人間手牌を優先して拡大する。
- 現在手番は色だけでなく「手番」ラベルでも表示する。
- 中央パネル上部に状態文を表示する。
  - `あなたの番です`
  - `CPUの手番です`
  - `流局しました`
- スマホでは人間の手牌を最優先で見やすくする。
- 押せない牌は見た目で分かるようにする。
- 文字は小さくしすぎない。
- CPUの処理が速すぎて分かりにくくならないよう、300msから600ms程度のdelayを想定する。
- UIは `gameState` を直接変更しない。
- 操作は `dispatchAction(state, action)` を呼ぶ設計にする。

## MVP-0.1ではやらないデザイン改善

- 派手なアニメーション
- 効果音
- 牌画像素材の導入
- 横画面専用UI
- 鳴き、リーチ、和了ボタン
- 詳細な点数棒表示
- AI思考表示
- テーマ切替
- 詳細なルール説明画面

## 面接後の次アクション

- 統合済みUIをスマホ幅で確認する。
- 人間手牌のタップ領域を実測する。
- 現在手番表示をより目立たせる。
- 大きい牌モードで13枚から14枚が自然に横スクロールできるか確認する。
- 高齢者向けに文言を「短く、具体的」に再点検する。
- コントラストとフォーカス表示を確認する。
- CPU進行delayが速すぎないか手動確認する。
